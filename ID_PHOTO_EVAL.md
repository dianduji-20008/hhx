# 证件照小程序 — 可行性评估

> 基于 2026年6月 的研讨，对证件照小程序的产品定位、技术方案、成本收入、合规要求进行全面评估。

## 一、产品定位

### 1.1 核心定位

**小商家和求职者的 AI 证件照工具** — 上传照片，AI 抠图换背景，选尺寸，下载无水印。

### 1.2 功能清单

| 功能 | 技术方案 | 是否需要服务端 |
|------|---------|--------------|
| AI 抠图换背景 | RMBG-2.0 / MODNet（商业 API） | 是（API 调用） |
| 选尺寸裁剪 | Canvas | 否（纯前端） |
| 换装 | SDXL + ControlNet（商业 API） | 是（API 调用） |
| 美颜（V1） | Canvas 磨皮/美白/瘦脸 | 否（纯前端） |
| 美颜（V2） | 人脸关键点 + 局部重绘（商业 API） | 是（API 调用） |
| 合规检测 | 规则引擎（像素/头部比例/背景色值） | 否 |
| A4 排版 | Canvas | 否 |
| 600dpi 输出 | Canvas | 否 |

### 1.3 不做的功能

| 功能 | 原因 |
|------|------|
| 回执 | 需要公安系统对接资质，个体户拿不到 |
| 去水印 | 法律风险极高，中国已明确禁止去除 AI 生成标识 |

### 1.4 回执解释

证件照回执是公安系统的审核凭证。拍完证件照后，照片上传到公安数据库审核，审核通过后给一个回执编号，办身份证/驾照/护照时用。

- 只有公安系统证件（身份证、驾照、护照）需要回执
- 考试报名、求职简历、签证、学生证等场景**不需要回执**，只需要合规电子照片
- 不需要回执的场景占证件照总需求的 **70%+**
- 每年考试报名 3亿+ 人次，68% 因照片不合格被驳回 → 痛点真实

## 三、品牌标识与 AI 标识

### 3.1 两种标识的区别

| | 品牌标识（我们的水印） | AI 生成标识（法律要求） |
|---|---|---|
| 是什么 | 图片角落的"槐花香"小字 | 标注"此图片经AI处理"的标识 |
| 谁加的 | 我们主动加的 | 法律要求必须加的 |
| 能去掉吗 | 付费可去（合法） | **不能去，去违法** |
| 位置 | 图片角落，视觉上不明显 | 显性（页面上标注）+ 隐性（元数据/数字水印） |

### 3.2 具体实现

- **免费版**：图片有"槐花香"品牌小字 + 页面标注"AI 生成"
- **付费版**：去掉"槐花香"品牌小字 + 页面仍标注"AI 生成"
- **AI 生成标识永远不能去**，这是法律红线，和品牌水印无关

付费点"去品牌标识"完全合法——去的是自己加的商业水印，不是法律要求的 AI 标识。

## 四、技术方案

### 4.1 商业 API vs 自部署

| 维度 | 商业 API | 自部署 |
|------|---------|--------|
| 启动成本 | 0（按量付费） | GPU 服务器 ~1,800 元/月起 |
| 运维 | 零 | 需要管理模型、监控、扩容 |
| 冷启动 | 5-30 秒（Replicate） | 无（模型常驻） |
| 延迟 | 200-500ms + 网络延迟 | 100-300ms（本地） |
| 数据合规 | 图片经过第三方服务器 | 图片在自己的服务器 |
| 扩展性 | 自动扩容 | 需手动加 GPU |
| 价格随量变化 | 量越大越贵（线性） | 量越大越便宜（固定成本摊薄） |

### 4.2 结论：全程用商业 API

**商业 API 不是过渡方案，而是最优方案。**

1. **利润率极高**：API 成本只占收入 2%，利润率 ~98%，量再大也划算
2. **合规更轻**：调 API 只需大模型登记（1-2 个月），自部署需大模型备案（5-10 个月）
3. **零运维**：省下来的精力用来做产品和获客

### 4.3 V1 商业 API 方案

| 功能 | API | 单次成本 |
|------|-----|---------|
| 抠图 | Replicate RMBG-2.0 | ~0.02 元 |
| 换装 | Replicate SDXL + ControlNet | ~0.03 元 |
| 美颜 | 前端 Canvas | 0 |
| 换背景 | 前端 Canvas | 0 |
| 裁剪/排版 | 前端 Canvas | 0 |

V1 单用户平均 GPU 成本：~0.05 元

### 4.4 分阶段演进

| 阶段 | 方案 | 理由 |
|------|------|------|
| V1（0-3个月） | 全商业 API | 用户量未知，先验证需求，零启动成本 |
| V2（3-6个月） | 混合方案 | 抠图自部署（CPU 即可），换装继续用 API |
| V3（6个月后） | 视情况决定 | 日均用户超 2000 后评估是否自部署 GPU |

## 六、合规要求

### 6.1 必须做的备案

证件照的换装、美颜、抠图换背景属于《深度合成管理规定》中的"人物图像生成或显著改变个人身份特征的编辑服务"，**必须备案**。

| 备案项 | 适用方案 | 周期 | 费用 |
|--------|---------|------|------|
| 算法备案（生成合成类） | 所有方案必须 | 2-4 个月 | 0 元（政府不收费），代办 1-3 万 |
| 大模型登记（调 API） | 调用第三方 API 时必须 | 1-2 个月 | 0 元，代办 0.5-1 万 |
| 大模型备案（自部署） | 自研/微调模型时必须 | 5-10 个月 | 0 元，代办 3-5 万 |

**无论自研还是调 API，应用方都是"服务提供者"，必须建立内容审核机制。**

### 6.2 内容审核

证件照场景审核极简：

| 审核项 | 方案 | 成本 |
|--------|------|------|
| 输入图片是否含人脸 | 腾讯云人脸检测 | ~0.001 元/次 |
| 是否色情/违规图片 | 百度/腾讯内容安全 API | ~0.002 元/次 |
| 是否是本人照片 | 用户上传时勾选"确认是本人照片" | 0 |
| 未成年人保护 | 用户协议中注明 | 0 |
| 输出结果审核 | 证件照输出内容高度可控，无需额外审核 | 0 |

实际审核成本：~0.003 元/次，可忽略。

### 6.3 用户反馈

法规要求设置"公众投诉、举报入口"。证件照场景下：

- **投诉**：需要（用户付费出问题、功能不好用）
- **举报**：不需要（用户上传自己的照片生成自己的证件照，不存在"举报别人"的场景）

备案材料写法：

> 本产品仅处理用户本人上传的个人照片，输出内容为证件照，不涉及公开内容发布，不存在第三方举报场景。已设置用户投诉反馈入口，48小时内响应。

具体实现：小程序"我的"页面加"意见反馈"按钮，用户提交后消息推到团队后台，团队处理并回复。不需要对接政府部门。

### 6.4 备案材料清单

| 材料 | 说明 |
|------|------|
| 营业执照 | 个体户即可 |
| 算法安全责任人 | 身份证+社保证明 |
| 算法安全自评估报告 | 核心文档，描述算法原理、风险防控、内容审核机制 |
| AI 生成内容标识方案 | 显性（页面上标注"AI 生成"）+ 隐性（元数据/数字水印） |
| 用户实名认证方案 | 手机号验证码登录 |
| 内容审核机制 | 接入内容安全 API 检测输入 |
| ICP 备案号 | 域名备案 |

### 6.5 小程序上架要求

2026 年起，微信等平台对涉及 AI 功能的小程序**强制要求提交算法备案号**，否则不予上架。备案需在开发阶段就启动，和开发并行。
